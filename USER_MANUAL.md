# ICM Calculator ユーザーマニュアル (高度機能版)

本ツールは、ポーカートーナメントにおける賞金期待値（ICM）に基づき、最適な意思決定を支援するための高度な計算エンジンです。

## 1. 主要な指標と定義

### ■ ICM (Independent Chip Model)
*   **定義**: 現在のチップスタックが、最終的な賞金の何%に相当するかを算出するモデル。
*   **計算**: Malmuth-Harvilleモデルを使用。スタック比率を「1位になる確率」とし、再帰的に全順位の確率を計算します。

### ■ Bubble Factor (BF)
*   **定義**: 期待値の損失（リスク）と上昇（リワード）の比率。
*   **計算**: `(EV_now - EV_lose) / (EV_win - EV_now)`
*   **解釈**: BF=2.0の場合、勝った時に得るEVの2倍のEVを、負けた時に失うことを意味します。

### ■ Blocker (カード除去効果)
*   **定義**: 自分が持っているカード（Heroのハンド）は、相手持っていないという事実を計算に反映すること。
*   **計算**: 自分が `Ah` を持っている場合、相手のレンジ内の `Ax` コンボを数学的に除外・調整します。

### ■ Risk Score (次BBリスク)
*   **定義**: 強制ベット（BB）によってスタックが削られるまで猶予を数値化した指標。
*   **計算**: `Orbit_Left / (Stack / Blinds_Total)`
*   **解釈**: 数値が高いほど「死の淵」が近く、よりアグレッシブな判断（Push等）が検討されるべき状況です。

---

## 2. 使い方 (CLI)

高度な分析を行うには `cli_advanced.py` を使用します。

### 実行コマンド例
```bash
python cli_advanced.py --hand AhKd --stacks 10000 5000 2000 --payouts 50 30 --hero 2 --villain 0
```

### 引数説明
- `--hand`: 自分のハンド (2文字+2文字, 例: `AsKs`)
- `--stacks`: 全プレイヤーのスタック
- `--payouts`: 賞金構造
- `--hero`: 自分のインデックス (0から開始)
- `--blinds`: 1ハンドあたりの合計ブラインド
- `--orb_left`: BBが回ってくるまでの残り人数

---

## 3. 計算の仕組み

1.  **レンジ展開**: 指定された相手レンジ（%）を 1326 通りの具体的なコンボに展開します。
2.  **Blocker適用**: Heroのハンドと衝突するコンボを削除します。
3.  **Equity算出**: 残ったコンボに対して、Heroハンドの勝率（Equity）を算出します。
4.  **EVマッピング**: 「勝ち」「負け」「フォールド」の3つのスタック状態に対してICM計算を行い、Equityで加重平均して最終的なEVを求めます。
